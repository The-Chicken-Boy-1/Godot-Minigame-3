## Devlog
During this lesson, I noticed that the instructions encouraged testing small parts of the code step by step instead of writing everything at once. Usually when I code, I tend to write a lot of lines before I stop to check if it works. Sometimes I even try to finish most of the program before testing it. While this can feel faster at first, it can also cause problems because if something breaks, it can be harder to figure out exactly where the mistake is.

After doing this lesson, I think testing small sections of code and using print() statements is actually a better way to program. It helps catch errors earlier and makes it easier to understand what each part of the code is doing. I think my old method was slowing me down sometimes because I would have to search through a lot of code to find one small mistake. In the future, I plan to test my code more often as I build it step by step. I think this approach will make debugging easier and help me write better programs overall.
